Linux Cheat Sheet（Linux小抄）

命令

  • pwd：打印当前进程的工作目录
  • hostname：打印主机名称
  • exit：退出当前登录的用户
  • man：跟一个命令名，调出此命令的操作手册，非常有用的命令
  • cd：跳转当前进程的工作目录，~代表用户目录
  • ls：展示当前目录中的内容
    • -l：竖着展示更多内容，1. 权限标记，2. ?，3. 所有者，4. 用户组，5. ?，6. 修改日期
    • -a：显示当前目录下的所有文件及文件夹包括隐藏文件
    • -al：-a -l的组合
    • -p：显示目录后的斜杠
  • ll：跟ls作用相同，显示方式不同
  • clear：清空终端的屏幕内容
  • sudo：在当前命令上使用一次管理员权限
  • !!：运行上一次输入的命令，感觉没什么用，因为按上键就行了
  • su：切换当前用户，不带参数切到root账户
  • adduser：添加一个账户
sudo adduser xxx
sudo adduser xxx g # 添加一个用户 并且指定用户组
  • deluser：删除一个账户：sudo deluser xxx
  • passwd：更改账户密码：sudo passwd xxx
  • groupadd：添加一个用户组：sudo groupadd xxx
su xxx # 切换到xxx用户
  • chown: 修改文件的所有者，可应用多个文件或者目录
    • -R：递归变更
sudo chown user:ug a.txt # 修改a.txt的所有者为user，用户组为ug
  • chmod: 修改文件的权限标记，可应用多个文件或者目录
    • -R：递归变更
    • 标记u：所有者，g：用户组，o：其他人，a：ugo的组合
    • +表示增加权限，- 表示取消权限，= 表示唯一设定权限
    • r 表示可读取，w 表示可写入，x 表示可执行，X 表示只有当该文件是个子目录或者该文件已经被设定过为可执行
sudo chown 646 a.txt # 修改a.txt的权限，其中646代表二进制的 110 100 110 又代表 rw- r-- rw-
sudo chown ugo+r # ugo增加r权限
sudo chown ug+r, o-w # ug增加r权限，o减去w权限
  • rm：删除文件
    • -r：递归
    • -f：强制删除
rm a.txt # 删除a.txt
rm dir/* # 删除dir目录下所有文件
rm ./*.cpp # 删除当前目录中以.cpp结尾的文件
rm -rf / # 删库跑路
  • mkdir：创建目录
  • touch：创建文件
  • cp|mv：拷贝或移动文件
cp|mv A B # 拷贝或移动A到B
  • find：查找文件名或目录名
    • -type ?：f查找文件
    • -name ?: 支持通配符的文件名
    • -iname ?: -name基础上忽略大小写
    • -not: 匹配条件反转
    • -maxdepth 1: 最大递归深度
    • -size +10k: 匹配文件大小+10k，-10k
    • -exec ... +: 执行下一个命令，在+号结束之前是命令的内容
find . -type f -name "*.cpp" # 查找当前目录下所有.cpp结尾的文件
  • grep：匹配文件内容
    • -i：匹配内容忽略大小写
    • -n：输出的时候带上对应的行号
grep "function" a.txt b.txt # 匹配所有a.txt b.txt出现function的行
grep -i "function" ./* # 匹配目录下所有文件，忽略匹配内容大小写
  • >：把命令结果输出到文件，ls > a.txt
  • |：串联多个命令，a | b
  • tee：读取输入流并输出到文件，tee a.txt
  • top：类似于资源管理器，实时显示系统状态，可以看进程列表
  • ps aux：抓取当前系统状态并显示
  • pgrep：匹配之后打印的是进程ID
  • kill：匹配之后打印的是进程ID
    • -l：显示kill支持的所有信号
kill -HUP pid
kill -9 pid   # 1=SIGHUP重新加载  9=SIGKILL强杀  15=SIGTERM正常停止
  • killall：杀掉所有匹配的进程，指定进程名（Command）
  • sudo systemctl start|stop|restart|reload|status xxx：操作服务，systemctl是执行/lib/systemd中的可执行文件实现服务启用停用的，脚本中通常需要自己实现逻辑判断是启动或停用，如果不存在对应的文件它会回退到/etc/init.d中寻找（兼容service命令的执行目录）。
systemctl status xxx # 检查某个服务状态
systemctl start mysql # 启动mysql
systemctl list-units # 列出所有可用的单元
systemctl list-unit-files --type=service # 列出所有服务
systemctl kill mysql # 杀死某个服务
systemctl get-default # 获取默认启动级别
systemctl set-default multi-user.target # 设置默认启动级别
systemctl enable mysql
  • crontab：做一个定时任务，通常用-e按照内容指示操作
    • -u user：不指定是自己，root可以指定其他人
    • -e：在文本编辑器中编写要执行的任务
    • -l：列出目前的定时任务
    • file：指定file代表要从file中读取定时任务

Linux中执行两种定时任务：1. 系统任务， 2. 用户任务

crontab -e # 编辑一个定时任务
crontab a.txt # 从a.txt中读取
  • ssh：ssh到一台主机
ssh 用户名@主机名:端口号
  • ping：icmq消息发送
  • ifconfig：用于显示或设置网络设备，类似于Win中的ipconfig

Ubuntu

  • dpkg：安装一个本地的deb包

权限

示例：-rwxr-x--x

其中第一个字符表示文件类型

文件类型

  • -：普通文件
  • d：目录
  • l：链接文件
  • b：表示设备文件里面的可供存储的接口设备
  • c：表示设备文件里面的穿行端口设备，比如键盘、鼠标

后面的字符都是权限标记，其分为三段：

  1. 所有者的权限
  2. 同一个用户组的权限
  3. 其它人或公共的权限

每段含有三个字符，其顺序固定，rwx代表拥有读、写、执行权限，r-x代表只有读和执行权限，没有写权限，占位符-代表没有对应的权限。

权限标记

  • -：没有权限
  • r：读权限
  • w：写权限
  • x：可执行权限，对于目录是进入目录的权限
  • s：SET位权限
  • t：粘滞位权限
